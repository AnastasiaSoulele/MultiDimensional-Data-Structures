\begin{enumerate}

\item Η μεταβλητή max_nodes καθορίζει μέχρι πόσες εγγραφές ή mbr έχει κάθε κόμβος(αναγκαίο για τον καθορισμό του μηχανισμού υπερχείλησης των κόμβων )
υπάρχουν 2 μεθόδοι για το split το απλό split και το linear split,για πειραματικούς λόγους.Η καλύτερη μέθοδος διαχωρισμών κόμβων επηρεάζει την ταχύτητα αλλά και την σωστή σημιουργία των mbr, καθώς το linear split δεν οδηγεί σε ασυνέχεις ενώ το απλό οδηγεί.
Όλες οι μεθόδοι ακολοθούν μια recursive λογική καθώς είναι εθκολότερος ο διαπερασμός τοθ δέτρου.

\item Πιο αναλυτικά για τις μεθόδους:
για την κλάση Node():
def add_entry(self, entry):
Εδώ αξίζει να σημιωθεί πως ανάλογα αν το entry είναι φύλλο ή όχι το βάζουμε στο κατάλληλο node,επαναυπολογίσουμε το τα mbr σωστά και κάνουμε split αν χρειάζεται.

\item split(self) και linear_split():
εδώ αξίαζει να σημιωθεί πώς στο linear_split υπολογίζουμε ως πιο άξονα θα γίνει το sort των δεδομένων στη λίστα ώστε το split να είναι πιο αποδοτικό.
time complexity:O(N) και για τα 2 αλλά το linear όπως αναφέραμε υπολογίζει καλύτερα τα mbr.

\item def compute_mbr(self):
Απλός επανυοπολισμός των mbr (κυριώς χρησιμοποιείται όταν κανουμε splitη χρειάζεται να επανυπολογίσουμε το mbr του node πάλι απο την αρχή)

\item def enlarg_rect:
υπολογισμός ενός καινούριου mbr απο 2 άλλα

\item Έπειτα η κλάση Rtree:
χρησιμοποιεί και τις προηγούμενες μεθόδους τοθ Νode για σωστή υολοποίηση στις μεθόδους τους.Μια απλήεξήγηση των μεθόδων

\item def insert(self, entry):
κάνει insert ένα entry sto κατάλληλο node με την βοήθεια της leaf_area και καλεί την balance_tree σε περίπτωση σπασίματος

 \item def balance_tree(self, node, new_node):
 Διαχειριζόμαστε την ειδική περίπτωση της ρίζας, και άμα δεν είναι ρίζα ανεβάζουμε αναδρομικά πάνω το σπάσιμο.

\item def leaf_area(self, node, rect):
 Βρίσκει την κατάλληλη περιοχή για την εισαγωγή του καινούριου σημείου με βάση τα mbr.

\item def query_rtree(self, node, name_range, min_awards):
Μια συννάρτηση αναζήτησης πυ βασίζεται πάνω στην αναζήτηση με βάση τα mbr.
Δηλαδή συγκρίνουμε το query που έχει ζητηθεί με τα mbr του δέντρου και επιστρέφουμε τις κατάλληλες περιοχές.

\item Τέλος κάποιες γενικές πληροφορίες για το r-tree:o χρόνος αναζήτησης του δέντρου είναι Ο(log(max_nodes)N) κατα μέσο όρο καθώς log(max_nodes )N είναι το βάθος του δέντρου, και ίσως χρειαστεί να διαπεράσεις max_nodes εγγραφές μέσα σε ένα node και ο χειρότερος Ο(Ν).Οπότε για μία καλή υλοποίηση του δέντρου πρέπει να αναλογιστούμε ποιος είναι ο καλύτερος αριθμός max_nodes μέσα σε ένα κόμβο το οποίο εξαρτάται και από τηνν πυκνότητα των δεδομένων, και για την κατασκευή του δέντρου την καλύτερη μέθοδο διάσπασης κόμβων καθώς καθορίζει την χρονική πολυπλοκότητα της δημιουργίας του δέντρου αλλά και την ακεραίοτητα των δεδομένων.

\end{enumerate}
